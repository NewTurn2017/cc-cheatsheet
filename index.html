Session Control
  • /init CLAUDE.md 자동 생성
  • /clear 대화 내용 초기화 (새 대화 시작)
  • /compact 대화 압축 — 토큰 절약, 긴 대화 시 필수
  • /compact [키워드] 키워드만 남기고 압축
  • /resume 중단 세션 이어서 재개
  • /rewind 이전 대화 지점으로 되돌림
Monitoring
  • /status 버전 / 모델 / 계정 상태
  • /status line 하단 상태 표시줄 토글
  • /context 토큰 사용 현황 출력
  • /cost 비용 확인
  • /doctor 설치 및 시스템 상태 진단
Interaction
  • /voice 음성으로 명령 (마이크 입력)
  • @파일명 특정 파일을 대화에 첨부
Tip: 대화가 길어지면 AI가 앞 내용을 잊습니다 (Context Rot). 주기적으로 /compact 또는 /clear로 정리하세요.

⚡ WAT Framework (핵심 구조)
Workflow → Agent → Tools
  • Workflow 자동화 과정 — 여러 단계를 연결한 작업 흐름
  • Agent AI 실행자 — 독립 메모리, 모델 선택 가능
  • Tools 실행 도구 — MCP, API, CLI 명령 등
핵심: Workflow가 전체 흐름, Agent가 각 단계 실행, Tools가 실제 도구
🔧 Built-in Tools (내장 도구)

Agent (Sub)서브에이전트 — 하위 AI를 생성해 병렬 작업
🔌 MCP & Skills (외부 연동 & 자동화 스킬)
MCP (Model Context Protocol) — AI와 외부 도구를 연결하는 표준 규약
  • API 연동으로 데이터 수집 및 동작 실행
  • 외부 서비스(DB, Slack 등)와 Claude를 실시간 연결
  • Claude Code 명령어로 MCP 서버 연결/해제
Webhook — 이벤트 발생 시 자동 호출되는 URL
  • 이벤트 기반 자동 실행 트리거 — 예: 커밋 시 빌드
  • 서명 검증으로 보안 강화 필수 — 위조 방지
  • Base64 스트림 — 이미지/파일을 텍스트로 인코딩해 전송
Skills (스킬) — 반복 작업을 자동화하는 레시피
  • 반복 작업을 시스템화하는 프롬프트 집합
  • .claude/skills/ 스킬 저장 위치
  • skill.md YAML Front Matter — 파일 상단 설정 영역
  • /명령어 슬래시로 스킬 호출
  • 자연어 트리거 말로 자동 발동 — "디자인해줘" 등
Sub-agents — 메인 AI가 만드는 하위 AI
  • 스킬 내에서 하위 에이전트로 작업 분할 위임
  • 예: 코드 리뷰어, 디버거, DB 검증기 등
💡 Key Concepts (핵심 개념)
Context WindowAI가 한 번에 읽을 수 있는 최대 텍스트 양
Tokens텍스트 처리 최소 단위 — 1단어 ≈ 1~2토큰
Context Rot대화가 길어져 AI 성능이 떨어지는 현상 — /compact로 해결
Self-Healing오류 발생 시 AI가 스스로 감지하고 자동 수정
Stateless각 실행이 독립 세션 — 이전 기억 없음
Worktree같은 저장소에서 여러 브랜치를 동시에 작업하는 공간
Headless화면 없이 브라우저 실행 — 자동화 테스트용
Embedding데이터를 숫자 벡터로 변환 — 유사도 검색에 사용
🔀 Git Essentials (버전 관리)
  • git clone 원격 저장소를 내 컴퓨터에 복제
  • git branch [name] 새 작업 브랜치 생성 — 독립 작업 공간
  • git checkout 브랜치 전환 — 작업 공간 이동
  • git add . 변경된 파일을 스테이징 — 커밋 준비
  • git commit -m 변경사항 저장 (스냅샷)
  • git push 로컬 커밋을 원격(GitHub)에 업로드
  • git pull 원격 변경사항을 내 컴퓨터에 반영
  • git merge 다른 브랜치의 변경사항 합치기
